One Agent, Two Brains 🧠

The same agent code, run two ways: with a cloud LLM (DeepSeek) and with a local SLM on your own laptop (Ollama). Follow along step by step.

The two projects

Inside the course repository, the Day-One folder contains two versions of the same agent:

XIAN-Training/Day-One/ ├── agent-deepseek/ ← Brain in the CLOUD (DeepSeek API) │ ├── first_agent.py terminal chat │ ├── streamlit_app.py web app │ ├── requirements.txt │ └── .env.example needs DEEPSEEK_API_KEY └── agent-local-slm/ ← Brain on YOUR LAPTOP (Ollama) ├── first_agent.py terminal chat ├── streamlit_app.py web app ├── requirements.txt └── .env.example just picks the model — no key!

Same LangChain code, same chat loop, same web UI. The only real difference is where the model lives:

☁️ agent-deepseek💻 agent-local-slm
Modeldeepseek-chat (huge, smart)qwen2.5:1.5b (small, fast)
Runs onDeepSeek's serversyour own machine via Ollama

Privacyquestions go to the cloudnothing leaves your laptop

Before you start (once per laptop)

Get the code and open it in VS Code:

git clone https://github.com/drbahja/Software-Engineering-for-Gen-AI.git
cd Software-Engineering-for-Gen-AI

💡 Already cloned it in an earlier lab? Just run git pull to get the latest version.

Project A — ☁️ Agent with DeepSeek (cloud)

1Go into the project and create its virtual environment

cd XIAN-Training/Day-One/agent-deepseek

# Windows
python -m venv .venv
.venv\Scripts\activate

# macOS / Linux
python3 -m venv .venv
source .venv/bin/activate

You should see (.venv) at the start of your terminal line.

2Install the libraries

pip install -r requirements.txt

⚠️ Slow in class? Use the Tsinghua mirror:
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple

3Add the API key

Copy .env.example to a new file named exactly .env, then paste the key your instructor gave you:

# .env
DEEPSEEK_API_KEY=sk-xxxxxxxxxxxxxxxx

⚠️ Never share the key or commit .env to GitHub — .gitignore already blocks it.

4Run the terminal version

python first_agent.py

Ask it something — "Who was the first computer scientist?" — then type exit to quit.

5Run the web version

streamlit run streamlit_app.py

Your browser opens at http://localhost:8501. Same agent, now with a text box and a button. Press Ctrl+C in the terminal to stop it.

Project B — 💻 Agent with a local SLM (Ollama)

Now the same agent, but the brain moves onto your laptop. Full background in Lab Part 5.

1Install Ollama and pull a small model

Install Ollama from ollama.com (see Lab Part 5 for laptop-size advice), then:

ollama pull qwen2.5:1.5b
ollama list        # the model should appear here

2Set up this project's environment

Open a new terminal (or deactivate the previous venv — each project has its own):

cd XIAN-Training/Day-One/agent-local-slm

# Windows
python -m venv .venv
.venv\Scripts\activate

# macOS / Linux
python3 -m venv .venv
source .venv/bin/activate

pip install -r requirements.txt

3Configure — no key needed!

# Windows
copy .env.example .env

# macOS / Linux
cp .env.example .env

The .env only chooses which local model to use (OLLAMA_MODEL=qwen2.5:1.5b). No secret, nothing to protect — that is the point of local models.

4Run the terminal version

python first_agent.py

Notice: answers appear without internet. Turn Wi-Fi off and try it — it still works.

5Run the web version

streamlit run streamlit_app.py

Same UI as Project A at http://localhost:8501 — but look at the caption under the title: the model is running 100% on this laptop.

What to compare during the demo 🔍

  • Quality — ask both the same question. The cloud model knows more and reasons better; the SLM is shallower but instant and free.
  • Speed — the SLM often answers faster (no network round-trip).
  • The code — open both first_agent.py files side by side. Only the model setup differs; the agent logic is identical. Swapping the brain is a one-line decision.

Troubleshooting

ProblemFix
ModuleNotFoundErrorIs (.venv) showing? Activate the venv of this project and re-run pip install -r requirements.txt.
DEEPSEEK_API_KEY not foundThe file must be named exactly .env, inside agent-deepseek/, key pasted with no spaces or quotes.
"Could not reach Ollama"Is the Ollama app running (llama icon)? Did ollama pull finish? Check with ollama list.
Port 8501 already in useAnother Streamlit is running — stop it with Ctrl+C, or run with --server.port 8502.
pip very slowUse the Tsinghua mirror (Project A, step 2).
