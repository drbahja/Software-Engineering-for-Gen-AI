Your Agent, 100% on Your Laptop 💻

No cloud, no API key, no internet: run the same first agent on a Small Language Model that lives on your own machine.

What is an SLM — and why run one locally?

A Small Language Model (SLM) is a compact version of models like DeepSeek — small enough to run on a normal laptop. The trade-off: less knowledge and weaker reasoning, but you gain four huge advantages:

  • Private — your questions never leave your machine
  • Free — no API key, no cost per token
  • Offline — works with no internet at all
  • Yours — nobody can turn it off or change it

We'll use Ollama — the simplest tool in existence for running models locally. One install, one command, done.

1Install Ollama

  1. Go to ollama.com and click Download.
  2. Windows: run the downloaded OllamaSetup.exe → click Install. That's the whole installer.
  3. macOS: open the downloaded zip → drag Ollama to Applications → open it once (a small llama icon appears in the menu bar).
  4. Verify: open a terminal and type ollama --version. A version number = success.

⚠️ Slow download in class?The installer and models are large. If the classroom connection struggles, the instructor has the installer and the model file on a USB stick — raise your hand instead of waiting.

2Pick a model that fits YOUR laptop

Rule of thumb: the model must fit comfortably in your RAM, or your laptop will feel heavy and slow. Check your RAM (Windows: Settings → System → About · macOS: → About This Mac), then choose:

Your laptopModel to pullSizeWhat to expect
Any laptop, even old (≤8 GB RAM)qwen2.5:0.5b~0.4 GBInstant answers, basic quality — great for testing
8 GB RAM (the safe default ✅)qwen2.5:1.5b~1 GBFast, surprisingly capable — our recommendation
16 GB RAMqwen2.5:3b~2 GBBetter reasoning, still smooth
16 GB+ with Apple Silicon / good GPUqwen2.5:7b or deepseek-r1:7b~5 GBGenuinely good; r1 shows its reasoning steps

Download your model (we'll use the safe default — if unsure, choose this one):

ollama pull qwen2.5:1.5b

Then have a quick chat with it directly in the terminal to prove it works:

ollama run qwen2.5:1.5b

Ask it something, then type /bye to exit.

💡 Why Qwen?Qwen (通义千问) is Alibaba's open model family — excellent in both English and Chinese, and among the best small models in the world. A local model that speaks your language!

3Your local API — and why it's secure

Here is the magic: while Ollama is running, it serves a local API that speaks the same language as OpenAI's API. Test it:

curl http://localhost:11434/v1/models

You should see JSON listing your model. Your laptop is now an AI server!

Why is this secure? Look at the address: localhost (= 127.0.0.1) means "this machine only". The API is not visible to the Wi-Fi network, the classroom, or the internet — only programs running on your own laptop can reach it. That's why no API key is needed.

  • ✅ localhost:11434 — private to your machine (the default; keep it)
  • ⚠️ Never bind Ollama to 0.0.0.0 (a setting some tutorials show) — that would open your AI to the whole network. If you ever need that, add authentication in front of it.

💡 Connect the dotsRemember the .env lesson: cloud APIs need secret keys because they're on the public internet. A localhost API is protected by the network itself. Different threat, different defence — that's security thinking.

4Create a NEW project (don't touch the old one!)

Professionals never experiment inside a working project. New experiment = new folder = new venv:

  1. In your genai-course folder, create a new folder: agent-local-slm.
  2. VS Code → File → Open Folder… → select it (open a new window if you want to keep the old project open too).
  3. Terminal → New Terminal, then create the environment:
# macOS / Linux
python3 -m venv .venv
source .venv/bin/activate

# Windows
python -m venv .venv
.venv\Scripts\activate
  1. Create requirements.txt with exactly these lines, then install:
langchain-openai>=0.3
langchain-core>=0.3
python-dotenv>=1.0
streamlit>=1.40
pip install -r requirements.txt

💡 NoticeWe install langchain-openai, not langchain-deepseek — because Ollama speaks the OpenAI-compatible dialect. The abstraction lesson strikes again.

5Change the code — 3 lines

Create first_agent.py and copy your Day-2 agent into it. Now compare the only part that changes:

# DeepSeek version (cloud)              # Local SLM version (your laptop)
from langchain_deepseek \
     import ChatDeepSeek                 from langchain_openai import ChatOpenAI

model = ChatDeepSeek(                    model = ChatOpenAI(
    api_key=deepseek_key,  # secret!         api_key="ollama",   # any text — nothing is secret
    model="deepseek-chat",                   model="qwen2.5:1.5b",
    temperature=0.7,                         base_url="http://localhost:11434/v1",
)                                            temperature=0.7,
                                         )

Three real changes: the import/class, the model name, and the base_url pointing at your own machine. Everything else — messages, invoke, the chat loop — is untouched.

Also create a .env file with one line, so changing model later needs no code edit:

OLLAMA_MODEL=qwen2.5:1.5b

and read it in the code: llm_name = os.getenv("OLLAMA_MODEL", "qwen2.5:1.5b")

Run it:

python first_agent.py

Chat with your fully-local agent. Unplug the Wi-Fi and chat again — it still works. 🤯

🏆 Exercise — cloud vs laptop showdown

You now have TWO agents: DeepSeek (cloud) and Qwen (local). Put them head to head:

  1. Ask both the same 3 questions: a fact ("capital of Australia?"), a reasoning puzzle ("If I have 3 apples and eat half of one…"), and a Chinese-language question.
  2. Score each answer: speed, correctness, detail. Where does the small model hold up? Where does it clearly lose?
  3. Swap models in .env (try qwen2.5:0.5b — pull it first) and feel the size/quality trade-off yourself.
  4. Discussion: when would a company choose a local SLM over a cloud LLM? (Think: privacy, cost at scale, offline factories, latency.)

Deliverable: your comparison table + one sentence: "I would use a local SLM when ______."

✅ What you just learned

You installed a model server, chose a model for your hardware, understood localhost security, and switched providers with 3 lines — cloud or laptop, the software engineering stays the same. That is the whole point of this course.
