Who Does What? 👥

AI projects fail without the right team. Learn the 13 key roles — then staff a real project yourself.

Step 1 — Meet the roles (15 min)

Keep this table open — it is your reference for the exercise:

RoleKey responsibilities
AI Product ManagerBridges technical and business teams; defines vision, translates problems, sets metrics
Industry Domain ExpertProvides subject-matter expertise, validates data and features
Project ManagerCoordinates scope, timelines, communication, and risk
AI ResearcherExplores novel models and learning techniques
Data ScientistAnalyzes data, builds models, extracts insights
Machine Learning EngineerBuilds scalable ML pipelines, deploys models
Responsible AI / Ethics LeadAudits models for fairness, ethics, and bias
UX DesignerDesigns intuitive AI interfaces and feedback loops
Software EngineerDevelops applications integrating AI models
Solution ArchitectDesigns system architecture and integration plan
MLOps ArchitectManages deployment, monitoring, and updates
Data EngineerBuilds ETL pipelines, structures and cleans data
Data ArchitectDesigns data infrastructure, governance policies

The confusing pairs — how to tell them apart

  • Data Scientist vs AI Researcher: the scientist applies known techniques to business data; the researcher invents new techniques.
  • ML Engineer vs MLOps Architect: the ML engineer builds the pipeline that trains/serves the model; MLOps keeps it running in production (monitoring, updates).
  • Solution Architect vs Data Engineer: the architect draws the whole system blueprint (cloud, integrations); the data engineer builds the data plumbing inside it.
  • AI Product Manager vs Project Manager: the product manager decides what to build and why; the project manager makes sure it gets built on time.

Worked examples

  • "Decide which customer complaints the AI should handle first, based on business value" → AI Product Manager (what to build & why = product decision).
  • "Check that the model's medical terminology matches how doctors actually write" → Industry Domain Expert (real-world subject knowledge).

🏆 Exercise — staff the chatbot project (20 min, in pairs)

Scenario: your company is launching an AI-powered customer-support chatbot to reduce wait times and improve satisfaction. It must understand inquiries, answer instantly, and escalate to a human when needed.

Task: assign one appropriate role from the table above to each task below. Write one short reason ("because…") next to each — the reason is where the learning happens.

#TaskRole + why
1Define the overall AI vision and align it with the business goals
2Design and train the chatbot's natural-language processing model
3Validate the customer complaint categories with real examples
4Set up the cloud infrastructure to store and process chatbot data
5Monitor the chatbot's accuracy and response time in production
6Ensure fairness and avoid bias in chatbot responses
7Build the data pipelines to collect, clean, and store customer chats
8Connect the chatbot to the company's CRM system
9Design how users interact with the chatbot and receive feedback
10Coordinate the project timeline, sprints, and risk-mitigation plans
